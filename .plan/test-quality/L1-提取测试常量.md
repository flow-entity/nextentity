# 任务 L1: 提取测试常量

## 优先级：低

## 问题描述
测试代码中存在多处 magic numbers，这些数字没有明确含义，降低代码可读性。

## 问题示例

### QueryBuilderIntegrationTest.java
```java
assertThat(employees).hasSize(12);  // 12 是什么？
assertThat(deptIds).hasSize(5);     // 为什么是 5？
assertThat(employee.getSalary()).isEqualTo(85000.0);  // 85000 从哪来的？
```

### 改进后
```java
private static final int TOTAL_TEST_EMPLOYEES = 12;
private static final int TOTAL_DEPARTMENTS = 5;
private static final double MAX_SALARY = 85000.0;

assertThat(employees).hasSize(TOTAL_TEST_EMPLOYEES);
assertThat(deptIds).hasSize(TOTAL_DEPARTMENTS);
assertThat(employee.getSalary()).isEqualTo(MAX_SALARY);
```

## 建议提取的常量

### 在 QueryBuilderIntegrationTest 中

```java
public class QueryBuilderIntegrationTest {
    // Employee 测试数据
    private static final int TOTAL_TEST_EMPLOYEES = 12;
    private static final double MIN_SALARY = 48000.0;
    private static final double MAX_SALARY = 85000.0;
    private static final long ALICE_EMPLOYEE_ID = 1L;
    private static final String ALICE_NAME = "Alice Johnson";

    // Department 测试数据
    private static final int TOTAL_DEPARTMENTS = 5;
    private static final int DISTINCT_DEPT_COUNT = 5;

    // 分页测试
    private static final int PAGE_SIZE = 5;
    private static final int OFFSET = 3;
}
```

### 在 CrudOperationsIntegrationTest 中

```java
public class CrudOperationsIntegrationTest {
    // 测试 ID 范围
    private static final long TEST_EMPLOYEE_START_ID = 100L;
    private static final long TEST_DEPT_START_ID = 100L;

    // 默认测试值
    private static final double DEFAULT_SALARY = 50000.0;
    private static final long DEFAULT_DEPT_ID = 1L;
}
```

## 何时需要提取常量

| 情况 | 建议 |
|------|------|
| 数字出现 2 次以上 | 必须提取 |
| 数字含义不明确 | 必须提取 |
| 数字可能变化 | 必须提取 |
| 数字是测试数据固有部分（如 ID=1） | 可选提取 |
| 数字是标准值（如 0, 1, 2） | 无需提取 |

## 执行步骤

1. 扫描测试文件识别 magic numbers
2. 评估每个数字是否需要提取
3. 在类顶部定义常量
4. 替换魔法数字为常量引用
5. 添加 JavaDoc 说明常量用途

## 验收标准

- 所有重复出现的数字被提取为常量
- 所有含义不明确的数字被提取为具名常量
- 常量命名清晰表达其用途
