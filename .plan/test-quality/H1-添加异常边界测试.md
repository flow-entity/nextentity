# 任务 H1: 添加异常和边界条件测试

## 优先级：高

## 问题描述
当前测试主要覆盖正常路径（happy path），缺少对异常情况、边界条件和错误输入的测试。这可能导致代码在遇到异常输入时行为不可预测。

## 影响文件

### 1. ExpressionBuilderImplTest.java
**缺失测试**：
- `eqIfNotNull` 已测试 null 和非 null 值（已覆盖）
- 缺少：null path、空字符串、非法类型转换测试

**建议添加**：
```java
@Test
void where_WithNullPath_ShouldHandleGracefully() {
    // given
    Path<Employee, String> nullPath = null;

    // when/then
    assertThatThrownBy(() -> whereImpl.where(nullPath))
        .isInstanceOf(NullPointerException.class);
}
```

### 2. QueryBuilderTest.java
**缺失测试**：
- 空结果集处理
- 无效查询条件
- limit/offset 边界值（0, 负数，Integer.MAX_VALUE）

**建议添加**：
```java
@Test
void limit_WithZero_ShouldReturnEmptyResult() {
    // when
    List<Employee> employees = queryBuilder.limit(0).getList();

    // then
    assertThat(employees).isEmpty();
}

@Test
void limit_WithNegativeValue_ShouldThrowException() {
    // when/then
    assertThatThrownBy(() -> queryBuilder.limit(-1))
        .isInstanceOf(IllegalArgumentException.class);
}
```

### 3. WhereImplTest.java
**缺失测试**：
- null 条件处理（部分已覆盖）
- 空集合 in/notIn 操作
- 嵌套条件优先级

**建议添加**：
```java
@Test
void where_WithEmptyInList_ShouldThrowException() {
    // given
    List<Long> emptyIds = Collections.emptyList();

    // when/then
    assertThatThrownBy(() ->
        whereImpl.where(Employee::getId).in(emptyIds))
        .isInstanceOf(IllegalArgumentException.class);
}
```

### 4. NumberOperatorImplTest.java
**缺失测试**：
- 除以零处理
- 数值溢出
- null 值操作

**建议添加**：
```java
@Test
void divide_WithZero_ShouldHandleDivisionByZero() {
    // when/then
    assertThatThrownBy(() -> operator.divide(0.0).eq(1000.0))
        .isInstanceOf(ArithmeticException.class);
}
```

## 执行计划

1. 分析每个测试类的被测类，识别可能的异常点
2. 为每个异常点添加对应的测试用例
3. 运行现有测试确保新增测试不影响原有功能
4. 生成覆盖率报告验证改进效果

## 验收标准

- 所有新增测试通过
- 核心类的行覆盖率提升至 90%+
- 分支覆盖率提升至 85%+
