# 任务 M2: 统一断言风格

## 优先级：中

## 问题描述
项目中同时存在两种断言风格：
1. **AssertJ** (推荐): `assertThat().isEqualTo()`, `assertThat().isTrue()`
2. **JUnit 4 风格**: `assertEquals()`, `assertTrue()`, `assertNull()`

混合使用两种风格降低了代码一致性，增加维护成本。

## 影响文件

### CrudOperationsIntegrationTest.java

当前使用 JUnit 4 风格断言：
```java
import static org.junit.jupiter.api.Assertions.*;

// 问题代码
assertEquals(1, employees.size());
assertEquals("Test User", employees.get(0).getName());
assertNotEquals(originalName, updated.getName());
assertTrue(after.isEmpty());
assertNull(inserted.getEmail());
```

### 建议替换为 AssertJ 风格

```java
import static org.assertj.core.api.Assertions.assertThat;

// 推荐写法
assertThat(employees).hasSize(1);
assertThat(employees.get(0).getName()).isEqualTo("Test User");
assertThat(updated.getName()).isNotEqualTo(originalName);
assertThat(after).isEmpty();
assertThat(inserted.getEmail()).isNull();
```

## AssertJ 优势

1. **流式 API**：更好的可读性
2. **丰富断言**：针对集合、字符串、日期等有专用断言
3. **更好错误信息**：失败时显示更详细的差异
4. **IDE 友好**：自动补全支持更好

## 迁移对照表

| JUnit 4 风格 | AssertJ 风格 |
|-------------|-------------|
| `assertEquals(expected, actual)` | `assertThat(actual).isEqualTo(expected)` |
| `assertEquals(expected, actual, delta)` | `assertThat(actual).isCloseTo(expected, within(delta))` |
| `assertTrue(condition)` | `assertThat(condition).isTrue()` |
| `assertFalse(condition)` | `assertThat(condition).isFalse()` |
| `assertNull(object)` | `assertThat(object).isNull()` |
| `assertNotNull(object)` | `assertThat(object).isNotNull()` |
| `assertSame(expected, actual)` | `assertThat(actual).isSameAs(expected)` |
| `assertNotSame(expected, actual)` | `assertThat(actual).isNotSameAs(expected)` |
| `assertThrows(Exception.class, () -> {...})` | `assertThatThrownBy(() -> {...}).isInstanceOf(Exception.class)` |
| `fail("message")` | `fail("message")` (same) |

## 执行步骤

1. 使用 IDE 全局搜索 `import static org.junit.jupiter.api.Assertions`
2. 逐个文件替换为 AssertJ import
3. 使用 IDE 重构功能或手动替换断言
4. 运行测试确保行为不变

## 自动化迁移脚本 (可选)

可使用 OpenRewrite 等工具自动迁移：
```yaml
# Maven
plugins:
  - openrewrite-maven-plugin
    configuration:
      recipes:
        - org.openrewrite.java.testing.junit5.UseAssertJ
```

## 验收标准

- 所有测试文件统一使用 AssertJ
- 移除所有 `import static org.junit.jupiter.api.Assertions.*`
- 代码风格检查通过
