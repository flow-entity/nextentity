# 任务 L2: 清理冗余测试注释

## 优先级：低

## 问题描述
当前测试注释采用三段式结构（Test objective/Test scenario/Expected result），对于简单测试过于冗长。

## 当前注释风格

```java
/**
 * Test objective: Verify that select with Class returns correct WhereStep.
 * Test scenario: Call select with a projection class.
 * Expected result: Returns WhereStep with SelectProjection.
 */
@Test
void select_WithClass_ShouldReturnWhereStep() {
    // when
    var result = queryBuilder.select(String.class);
    // then
    assertThat(result).isNotNull();
    ...
}
```

## 问题分析

1. **方法名已表达测试意图**：`select_WithClass_ShouldReturnWhereStep` 已经清晰说明测试内容
2. **注释与代码重复**：注释描述的和代码实际做的完全一样
3. **维护成本**：修改测试时需要同步更新注释

## 建议的注释规范

### 简单测试 - 无需注释

```java
@Test
void select_WithClass_ShouldReturnWhereStep() {
    var result = queryBuilder.select(String.class);
    assertThat(result).isNotNull();
}
```

### 复杂测试 - 保留业务背景注释

```java
/**
 * 验证 DISTINCT 查询在大数据量下的性能。
 * 使用 1000 条记录测试查询响应时间应在 100ms 内。
 */
@Test
void selectDistinct_WithLargeDataset_ShouldCompleteInTime() {
    // given - 1000 employees
    insertTestEmployees(1000);

    // when
    long startTime = System.nanoTime();
    List<String> names = queryBuilder.selectDistinct(Employee::getName).getList();
    long elapsed = TimeUnit.NANOSECONDS.toMillis(System.nanoTime() - startTime);

    // then
    assertThat(elapsed).isLessThan(100);
}
```

### Bug 修复测试 - 保留问题链接

```java
/**
 * 修复 issue #123: 删除不存在实体时未抛出异常。
 */
@Test
void delete_NonExistentEntity_ShouldThrowException() {
    ...
}
```

## 注释原则

| 测试类型 | 注释策略 |
|---------|---------|
| 简单单元测试 | 无需注释，方法名自解释 |
| 集成测试 | 简要说明测试目的 |
| 性能测试 | 说明性能指标和阈值 |
| Bug 修复 | 保留 issue 链接和背景 |
| 复杂业务逻辑 | 说明业务规则 |

## 执行步骤

1. 审查所有测试注释
2. 移除简单测试的模板化注释
3. 为复杂测试添加有意义的背景注释
4. 更新项目测试规范文档

## 验收标准

- 简单测试的注释减少 50%+
- 保留的注释都包含有价值的信息
- 新增测试遵循注释规范
